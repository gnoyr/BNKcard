# BNK 부산은행 인증/마이페이지 테스트 시나리오 명세서

> 작성 기준: 실제 프로젝트 소스코드 (`AuthService`, `UserService`, `AuthController`, `UserController`) 직접 분석  
> 테스트 방식: **JUnit 5 + Mockito** (`MockMvcBuilders.standaloneSetup`, Spring 컨텍스트 없음)  
> 실행 방법: STS → Run As → **JUnit Test**

---

## 파일 위치 (src/test/java/com/bnk/domain/)

```
auth/
  service/    AuthServiceTest.java     ← @ExtendWith(MockitoExtension.class)
  controller/ AuthControllerTest.java  ← standaloneSetup MockMvc
user/
  service/    UserServiceTest.java     ← @ExtendWith(MockitoExtension.class)
  controller/ UserControllerTest.java  ← standaloneSetup MockMvc + 커스텀 ArgumentResolver
```

---

## 공통 설계 원칙

| 항목 | 내용 |
|------|------|
| 토큰 저장소 | `MemoryTokenStore` (Redis 아님) |
| 이메일 서비스 | `sendVerificationEmail()` / `sendPasswordResetEmail()` |
| 토큰 키 | `email:verify:{email}` · `email:verified:{email}` · `pw:reset:{uuid}` |
| Controller 테스트 | `MockMvcBuilders.standaloneSetup()` — Spring 부팅 없음 |
| 인증 주입 | 커스텀 `HandlerMethodArgumentResolver` → `CustomUserDetails` 직접 반환 |
| CSRF | `SecurityConfig`에서 비활성화 → 테스트에서도 불필요 |

## TC-01 이메일 인증코드 발송

**API** `POST /api/auth/send-verify-code`  
**Service** `AuthService.sendVerifyCode()`  
**토큰 키** `email:verify:{email}` (TTL 10분)

| 번호 | 구분 | 시나리오 | 입력 | 기대 결과 | HTTP |
|------|------|----------|------|-----------|------|
| 01-1 | 정상 | 미가입 이메일 → 코드 저장 + 이메일 발송 | `hong@bnk.co.kr` (미가입) | `tokenStore.set("email:verify:...", code, 10L)` 호출<br>`emailService.sendVerificationEmail()` 호출 | 200 |
| 01-2 | 실패 | 이미 가입된 이메일 | `dup@bnk.co.kr` (가입됨) | `DUPLICATE_EMAIL` (U002) | 409 |
| 01-3 | 실패 | 이메일 형식 오류 | `notAnEmail` | Bean Validation 실패 | 400 |

**검증 포인트**
- `existsByEmail()` 쿼리는 `deleted_yn` 무관 (탈퇴 후 재가입 방지)
- 코드는 UUID 기반 6자리 대문자 (`UUID.randomUUID().replaceAll("-","").substring(0,6).toUpperCase()`)

## TC-03 회원가입

**API** `POST /api/auth/signup`  
**Service** `AuthService.signup()`  
**전제조건** `email:verified:{email}` = "Y" 필수

| 번호 | 구분 | 시나리오 | 입력 | 기대 결과 | HTTP |
|------|------|----------|------|-----------|------|
| 03-1 | 정상 | 이메일 인증 완료 후 정상 가입 | 모든 필드 유효 + 필수 약관 동의 | `userMapper.insertUser()` 호출<br>`updateEmailVerified("Y")` 즉시 호출<br>`tokenStore.delete("email:verified:...")` 호출 | 201 + userId |
| 03-2 | 실패 | 이메일 인증 미완료 | verified 플래그 없음 | `EMAIL_NOT_VERIFIED` (U007) | 403 |
| 03-3 | 실패 | 휴대폰 번호 중복 | 기존 가입자와 동일 phone | `DUPLICATE_PHONE` (U010) | 409 |
| 03-4 | 실패 | 필수 약관 미동의 | requiredYn='Y' 약관 ID 누락 | `REQUIRED_TERMS_NOT_AGREED` (T002) | 400 |
| 03-5 | 실패 | 이름 필드 누락 | `name: null` | Bean Validation 실패 | 400 |

**검증 포인트**
- 전화번호 포맷 변환 후 중복 체크: `MaskingUtil.formatPhone("01012345678")` → `"010-1234-5678"`
- 이메일 미완료: `EMAIL_NOT_VERIFIED` (회원가입) vs `VERIFY_TOKEN_INVALID` (코드 확인) — **에러코드 다름**

## TC-04 로그인

**API** `POST /api/auth/login`  
**Service** `AuthService.login()`  
**성공 시** Access Token + Refresh Token HttpOnly 쿠키 발급, 로그인 이력 INSERT

| 번호 | 구분 | 시나리오 | 입력 | 기대 결과 | HTTP |
|------|------|----------|------|-----------|------|
| 04-1 | 정상 | 올바른 이메일·비밀번호 | 정상 ACTIVE 계정 | Set-Cookie: `access_token`, `refresh_token`<br>message: "로그인에 성공했습니다." | 200 |
| 04-2 | 실패 | 없는 이메일 | `ghost@bnk.co.kr` | `USER_NOT_FOUND` (U001) | 404 |
| 04-3 | 실패 | 비밀번호 불일치 | 틀린 password | `INVALID_PASSWORD` (U003)<br>`incrementLoginFailCount()` 호출 | 400 |
| 04-4 | 실패 | 잠금 계정 | status=LOCKED, lockedUntil 미래 | `ACCOUNT_LOCKED` (U004) | 423 |
| 04-5 | 실패 | 탈퇴 계정 | status=WITHDRAWN | `ACCOUNT_WITHDRAWN` (U006) | 403 |
| 04-6 | 경계 | 비밀번호 5회 실패 | loginFailCount=4 → 1회 추가 | `updateLockedUntil()` 호출 (30분 잠금) | 400 |
| 04-7 | 정상 | 실패 횟수 있을 때 로그인 성공 | loginFailCount=2, 올바른 비밀번호 | `resetLoginFailCount()` 호출 | 200 |
| 04-8 | 실패 | 이메일 형식 오류 | `bad` | Bean Validation 실패 | 400 |
| 04-9 | 실패 | password 필드 누락 | `{email}` 만 전송 | Bean Validation 실패 | 400 |

**검증 포인트**
- `MAX_LOGIN_FAIL = 5`, `LOCK_DURATION_MIN = 30`
- IP주소·User-Agent → `LOGIN_HISTORIES` INSERT (성공·실패 모두)
- 트랜잭션: `noRollbackFor = BusinessException.class` → 실패 횟수 증가가 롤백되지 않음

## TC-09 비밀번호 재설정

**API** `POST /api/auth/reset-password`  
**Service** `AuthService.resetPassword()`  
**검증 순서** ① newPw == confirm → ② tokenStore 조회 → ③ updatePassword + revokeAll + token 삭제

| 번호 | 구분 | 시나리오 | 입력 | 기대 결과 | HTTP |
|------|------|----------|------|-----------|------|
| 09-1 | 정상 | 유효 토큰 + 비밀번호 일치 | 유효 token, new=confirm | `userMapper.updatePassword()` 호출<br>`userSessionMapper.revokeAllByUserId()` 호출 (전 기기 로그아웃)<br>`tokenStore.delete("pw:reset:...")` 호출 | 200 |
| 09-2 | 실패 | 비밀번호 확인 불일치 | new ≠ confirm | `PASSWORD_CONFIRM_MISMATCH` (U009) | 400 |
| 09-3 | 실패 | 만료·없는 토큰 | tokenStore → null | `VERIFY_TOKEN_INVALID` (U008) | 400 |
| 09-4 | 실패 | token 필드 누락 | `{newPassword, newPasswordConfirm}` | Bean Validation 실패 | 400 |

> ⚠️ **주의**: `VERIFY_TOKEN_INVALID` (U008) → `HttpStatus.BAD_REQUEST` **(400)** — 401 아님
